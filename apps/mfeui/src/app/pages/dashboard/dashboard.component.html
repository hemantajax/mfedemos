About Micro Frontends Architecture

This dashboard serves as the host application for managing multiple independent micro frontend applications. Each MFE can be developed, tested, and deployed independently while seamlessly integrating into this unified shell.

Independent Development Each MFE is a separate Nx project with its own dependencies and build process.
Autonomous Deployment Deploy MFEs independently without affecting other modules.
Team Autonomy Different teams can own and maintain different MFEs.

Quick Start Guide
Creating a New MFE:
  1. Generate a new Angular app: nx g @nx/angular:app my-mfe
  2. Configure Module Federation in the new app's webpack config
  3. Update the routes in this host application
  4. Start the MFE on its designated port
Available Ports:

Each MFE is assigned a specific port for local development:

  • 4200 - Host Application (this app)
  • 4201-4208 - Remote MFE Applications
